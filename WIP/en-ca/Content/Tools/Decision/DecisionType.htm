Configure decision types

The Decision Type entity stores the set of decisions that can be made at your organization. These decision types are used in business process flows.

Associate a Decision Type to a Transition Path to get the information of documentation to be generated on each decision. If specified on a transition path, a decision record is created in the corresponding <EntityName> Decision entity at this transition. If an approval process is required for the transition, a Decision Type must be specified.

Note: The approval process requires a Decision Type. For an approval process to kick in, a Decision Type and Approval Type must be set in a Transition Path record. See Configure a transition path.

Many decision types are pre-configured in GCPM. Some of the decision types for projects include:

  • Project approval

  • Project assessment approved
  • Project assessment ready for recommendation
  • Approve project assessment
  • ...and many others.

To configure a new decision type: 

  1. Navigate to Business Administration > Decision > Decision Types. Create a new record.

  2. Fill in the fields using onscreen help and the notes below: 
    • Name (EN / FR): Type a name that describes the decision.
    • Document configuration: Choose an existing CFG Document record. Leave empty if no document needs to be made for this decision.
    • Document Template: Choose an existing CFG Document Template record. If you entered a CFG Document but leave this field empty, only the XML snapshot will be generated. If the Document Template is selected, a human readable document will be generated such as Word or PDF depending on the configuration.
    • System generated documents: To generate multiple document records for a decision, add more Decision Type Document records to the grid.
  3. Save and close the record.
  4. A document record will be created and appended to the decision and to the decision's parent entity.

  5. If there is an approval process to the decision, the document will be created at the final approval and appended to that final approval decision record as well.

  6. To append a document to a decision, the record has to be saved first.
    Note: The Save button has to be used. Do not use Save & Close. Once closed, the record can only be opened through a decision history).

  7. Once the record is saved, documents can be added to the subgrid at the bottom of the page.

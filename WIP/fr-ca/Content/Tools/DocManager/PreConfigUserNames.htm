Configurer les noms d'utilisateur pour le système de gestion de documents

Avant de configurer le plugiciel de gestion de documents, les enregistrements de l'utilisateur CRM doivent être remplis avec les noms d'utilisateurs du système de gestion des documents (SGD) sélectionné.

  1. Ouvrez un enregistrement d'utilisateur et ajoutez le nom d'utilisateur SharePoint ou GCdocs dans le champ prévu à cet effet.

  2. Le plugiciel vérifie ces champs pour déterminer l'utilisateur qui appelle, pour empêcher les utilisateurs de CRM d'usurper l'identité d'un autre utilisateur dans le SGD externe.
  3. Mise en garde : Un utilisateur de CRM titulaire de droits d'usurpation d'identité peut adopter l'identité d'un autre utilisateur lorsqu'il interagit avec le SGD externe au moyen de l'interface API du plugiciel. Soyez prudent lorsque vous attribuez des droits d'usurpation d'identité.
  4. Enregistrez et fermez l'enregistrement.

Remarques :

SharePoint :
  • La mise en œuvre du plugiciel du connecteur de SharePoint n'utilise par le champ Nom d'utilisateur de SharePoint dans l'enregistrement de l'utilisateur.
  • Cela s'explique par le fait que l'usurpation d'identité côté client est impossible au moyen de REST API de SharePoint.
  • Cependant, vous voudrez peut-être vous servir de ce champ dans votre implémentation locale du mécanisme d'authentification SharePoint du plugiciel.
GCDocs
  • Le connecteur GCdocs récupérera le nom de l'utilisateur qui appelle à partir du champ Nom d'utilisateur de GCdocs dans l'enregistrement des utilisateurs CRM.
  • Si ce champ est vide, le connecteur prend implicitement la valeur du champ d'utilisateur CRM.
  • Il convient de voir à ce que les noms d'utilisateurs soient identiques pour CRM et GCdocs.

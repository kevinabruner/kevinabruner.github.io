À propos du plugiciel verouillage

Le plugiciel verrouillage est utilisé pour verrouiller un enregistrement basé sur un ou plusieurs valeurs de champ. Une hiérarchie des champs peut être configurée afin que les enregistrements enfants soient verrouillés par l'enregistrement parent. Par exemple, lorsqu'un projet est fermé, le projet ainsi que tous les enregistrements de débours sont verrouiller.

L'objectif du plugiciel verrouillage est de prévenir:

  • La modification d'un enregistrement verrouillé ou de ses enregistrements enfants ou petit-enfants.
  • La suppression d'un enregistrement verrouiller ou de ses enregistrements enfants ou petit-enfants.
  • L'ajout d'un enregistrement enfant à un enregistrement parent verrouillé.

Il est possible de verrouiller certains champs dans un enregistrement, mais de laisser les autres déverrouillés. Le verrouillage est établi dans une entité racine et la configuration détermine la façon dont les entités enfants et petit-enfants sont verrouillées.

Exemples :

  • Configurer un enregistrement de réclamation et des enregistrements d'un élément de réclamation lié à un verrouillage lorsque l'état est réglé à « Compléter ».
  • Configurer les sous-catégories de risque et les enregistrements d'élément afin qu'ils se verrouillent lorsque catégorie de risque parent est verrouillée.
Entité CFG Verrouillage

L'enregistrement CFG Verrouillage est une configuration principale pour le verrouillage des champs et il contient des grilles d'enregistrement Verrouillage associés. C'est une hiérarchie autoréférentielle, afin de vous aidez à établir des relations parent-enfant. Par exemple, si un projet est verrouillé, le flux de trésorerie et les élément de coût (lesquels sont les enregistrements enfants du projet) peuvent aussi être verrouillés.

Pour verrouiller un champ, vous devez créer un enregistrement CFG Verrouillage pour chaque entité racine et toutes les entités enfants et petit-enfants qui contiennent des champs à verrouiller. Un entité racine peut avoir plus de un enregistrement CFG Verrouillage, pourvu que chacun possède un champ verrouillage différent.

Les enregistrements CFG Verrouillage déterminent le champ verrouillage, la relation avec l'entité parent (en utilisant les champs: Configuration parent verrouillage, Nom du champ parent et Nom de la relation parent) et (si le type de champ verrouillage = deux options) enregistrent la liste des champs à être verrouillé (Liste des champs verrouillés).

Les Champs de verrouillage (egcs_lockfield) sont des champs qui déclenchent le verrouillage sur un enregistrement ou sur des enregistrements enfants ou petit-enfants. Un enregistrement racine ou parent peut avoir tous ces trois types de champs verrouillés. Les enregistrements CFG Verrouillage enfants peuvent seulement avoir le champ verrouillé « Deux options ».

Types de champ verrouillé supportés:

  • Deux options  (Oui / Non)
  • Chercher (racine seulement)
  • Groupe d’options (racine seulement)

Types de champs:

  • Le type Champ de verrouillage détermine si la grille Champ de verrouillage est affiché, ainsi que la ressource Web pour la sélection de champ.
  • Si le type de champ de verrouillage = deux options, sélectionnez la liste de champs à verrouiller.
    Les champs verrouillés sont enregistrés automatiquement dans le Sélecteur de champ.
  • Si le type de champ de verrouillage = recherche ou ensemble d'options, les valeurs du verrouillage et les champs correspondants à être verrouillés sont stockés dans les enregistrements de CFG Champ verrouillage.
Entité CFG Champ verouillage

L'entité de configuration CFG Champ verrouillage possède des enregistrements avec une valeur donnée de la recherche ou de l'ensemble d'options de l'entité ciblée et la liste de champs qui seront verrouillé si cette valeur est la valeur sélectionnée dans l'enregistrement qui est vérifié.

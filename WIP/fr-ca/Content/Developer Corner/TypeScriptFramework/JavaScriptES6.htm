JavaScript ES6

Certains des changements dans JavaScript EcmaScript6 (ES6) sont énumérés ci-dessous. Pour de plus amples renseignements sur JavaScript ES6 vous référez à http://es6-features.org/

Modules

JavaScript passe de l'espace de nommage général vers un une structure organisationnelle modulaire.

Il a un point d'entrée « principal » à la méthode statique principale de l'Application de console C#.

Un module JavaScript est une fichier JavaScript. Pour qu'un fichier JavaScript soit considérer comme un module, il doit exporter des fonctions. Par conséquent, tous les fichiers JavaScript sont des modules.

Modules - Exportation

Même sur un fichier JavaScript est un module, aucun des codes à l'intérieur du module peut être utilisé à moins qu'ils soient exportés. Par conséquent, les codes non exportés sont privés au module lui-même.

Ce qui peut être exporté

Remarque:
Variables

export var num = 5;
Objets

export { prop1: value1, prop2: value2, …propN: valueN };
Fonctions

export function functionName() { … }
Classes

export class Square extends Shape { … }
Modules

export * as Common from “common”

export { SDK } from “common”
Défaut

Seulement une exportation par défaut peut exister dans un module; aucun nom requis.

Peut être n'importe quelle exportation de type ci-dessus à l'exception des modules.

Exemple: export default function() { }

Modules - Importation

Les modules peuvent être importé de deux façon, mais les éléments autres que les modules ne peuvent pas être importé de cette façon.

  • Importer le module entier enveloppé par l'alias
  • Importation * comme Commun de « commun »;
  • Common.SDK.retrieveRecord();
  • Les parties d'importation du module par nom de module
  • importation { SDK } de « commun »
  • SDK.retrieveRecord();

Il n'est pas nécessaire d'inclure l'extension dans le chemin du fichier.

Classes

Le code est organisé comme les langues de programmation orientée objet comme Java ou C#.

Une syntaxe semblable à la classe a été ajoutée dans ES6 afin de déclarer les classes dans JavaScript et permettre l'héritage de classes.

Cette syntaxe facilite la lisibilité, mais JavaScript reste un héritage prototype et plus fonctionnel que la programmation orientée objet.

Les classes peuvent avoir des méthodes et des propriétés statiques et une instance des méthodes et des propriétés.

class Square extends Shape {

sides = 4;

constructor() {

super();

}

}

Fonctions des flèches

Il y a une nouvelle syntaxe pour les fonctions de déclaration, mais elles ne sont pas les mêmes au niveau sémantique. La portée et le contexte sont différent. Le terme « ceci » se réfère à différents éléments.

  • Avant :
  • fonction() { }
  • function(param1) { }
  • function(param1, param2, …paramN) { }
  • function(param1, param2, …paramN) { return expression; }
  • Function(param1) { }
  • « ceci » fait référence à la fonction elle-même
  • Après :
  • () => { }
  • param1 => { }
  • (param1, param2, …paramN) => { }
  • (param1, param2, …paramN) => expression;
  • (param1, …rest) => { }
  • « ceci » fait référence à l'objet ou à la fonction qui agit comme contenant

Évolution du code asynchrone

1. - Rappels (archaïque)

SDK.retrieveRecord(“egcs_sg_clients”, id, client => {

SDK.retrieveRecord(“accounts”, id, account => {

// do stuff

});

});

2. - Promesses (primitif)

SDK.retrieveRecord(“egcs_sg_clients”, id)

.then(client => {

return SDK.retrieveRecord(“accounts”, id);

})

.then(account => { // do stuff });

3. - Asynchrone ou en attente => { // do stuff });

var client     = await SDK.retrieveRecord(“egcs_sg_clients”, id);

var account = await SDK.retrieveRecord(“accounts”, id);

// do stuff

Déclaration de variables

Maintenant il y a trois façons de déclarer des variables dans JavaScript: var, let et const.

Elles ont chacune leurs propres caractéristiques :

  • - Promesses (primitive)
  • var
  • Accessible à l'intérieur de la fonction et sont renvoyé au sommet.
  • let
  • Accessible à l'intérieur des crochets dans lesquels elle a été déclarée.
  • const
  • Pareille mais en lecture seulement; elle ne peut pas être attribuée après avoir été déclarée.

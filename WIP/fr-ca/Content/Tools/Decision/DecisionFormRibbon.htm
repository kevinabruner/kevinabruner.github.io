Ajouter des boutons de transition aux rubans d'un formulaire

Pour ajouter des boutons de décision et de statut au ruban d'un formulaire :

    1. Préparez-vous à travailler avec Ribbon Workbench. Créer une solution uniquement avec le formulaire [ENTITÉ].
    2. Ouvrez Ribbon Workbench (atelier du ruban) dans la XRM Toolbox (boîte à outils XRM).
    3. Ajouter un bouton de menu horizontal.
      1. Dans l'ensemble de sections du formulaire, cliquez-glisser le bouton de menu horizontal à partir de la boîte à outils à gauche.
      2. Insérez une section de menu.
      3. Dans cette section de menu, cliquez-glisser un autre bouton menu horizontal et deux boutons. Remplissez les noms en cliquant sur chaque des boutons et sur le bouton menu horizontal.
    4. Ajoutez des commandes pour load (charger), loadTransitions (charger les transitions), makeTransition (faire une transition), transitionHisotry (historique de la transition) et decisionHistory (historique de la décision).
      1. Changez le nom de l'entité (souligné en rouge dans la capture d'écran ci-dessous) pour le remplacer par le nom de l'entité vers laquelle vous ajoutez la transition.
      2. Pour ajouter une commande, utilisez le symbole +.
      3. Créez une commande load (chargement). Changez l'ID de commande.

      4. Ajoutez une Display Rule (règle d'affichage) pour cacher le bouton de transition lorsque le formulaire est en mode create (création).
        Cliquez sur Add step et ajoutez un FormStateRule with the state Existing.

      5. Add an Enable Rule to the command as well. This will load the scripts on the loading of the page.
        Change the id as well as the name of the entity. Click Add Step and choose Custom Rule.
        FunctionName is EGCS.Main.Command.loadFormCommand, Library is $webresource:egcs_/shared/js/main.js.

      6. Add 2 Parameters, one of type CRM Parameter with the value PrimaryControl and one of type String with the value egcs_/entities/[ENTITY] /ribbons/form.

    5. Créer une commande loadTransitions .

      1. Changer l'ID de commande pour tenir compte de l'entité.
      2. Cliquez sur Add Action (ajouter une action) pour créer une action JavaScript personnalisée. La Library (Bibliothèque) est $webresource:egcs_/shared/js/main.js and Function Name is EGCS.Main.Command.[ENTITY].onLoadTransitionsMenu
      3. Cliquez sur Add Parameter (ajouter un paramètre) pour ajouter un paramètre CRM dont la valeur est établie à CommandProperties.
    6. Créer un makeTransitioncommand.

      1. Change the command Id to reflect the entity.

      2. Click Add Action to create a custom Javascript action. Library is $webresource:egcs_/shared/js/main.js and Function Name is EGCS.Main.Command.[ENTITY].onClickMakeTransition.
      3. Click Add Parameter to add a Crm Parameter with the value CommandProperties.
    7. Créer un transitionHistory command.

      1. Change the command Id to reflect the entity.

      2. Click Add Action to create a custom Javascript action. Library is $webresource:egcs_/shared/js/main.js and Function Name is EGCS.Main.Command.[ENTITY].onClickTransitionHistory.

    8. Créer un decisionHistory command.

      1. Change the command Id to reflect the entity.

      2. Click Add Action to create a custom Javascript action. Library is $webresource:egcs_/shared/js/main.js and Function Name is EGCS.Main.Command.[ENTITY].onClickDecisionHistory.

À propos des programmes

Les programmes sont l'outil de configuration central pour la plupart des fonctionnalités de GPSC. Ils sont configurés par les « super utilisateurs » opérationnels et ne sont pas visibles aux agents. Les programmes sont structurés dans une hiérarchie: programme, sous-programme, volet, etc. et peuvent avoir de multiples admissions.

Le GPSC utilise des modèles pour de nombreuses fonctionnalités: demandes, questions d'évaluation, conditions de débours et autres. Les modèles sont configurés avec les données appropriées qui sont copiées aux nouveaux projets dans le programme. Consultez À propose des fonctions qui utilisent des modèles.

Les flux de processus opérationnel gère le cycle de vie du projet (par exemple, admission de la demande, évaluation, décision, surveillance, fermeture). Les flux de processus opérationnel sont intégrés avec d'autres outils de GPSC comme les modèles de transition, les décisions, le cadre de mesure et les ensembles d'autorité. Consultez Échantillons de flux de processus d'affaires de la GPSC pour de plus amples détails.

Les deux programmes d'échantillon livrés avec le MVP du GPSC sont conçu pour présenter un nombre de fonctionnalités pour gérer le cycle de ive du projet:

  1. Programme de financement de base (simple)
  2. Stratégie pour les femmes en entrepreneuriat (hiérarchie de programme plus complexe avec de multiples admissions).
Données clés saisies dans les enregistrements de programme
 
Catégorie Détails

Données de déclaration de rapports
  • Données de base du programme, par exemple, nom, dates clés, programmes enfants, etc.
  • Génère facilement des rapports standards (par exemple, divulgation proactive).

Données de configuration
  • De nombreuses fonctionnalités sont configurées en utilisant des modèles.
  • Les données de modèle sont utilisées pour créer ou gérer les projets, les débours, les modifications et les documents.
  • Les précédents modèles stockés sur le programme pour faciliter le rapport sur l'historique.

Opportunité de financement
  • Opportunités de financement (admissions) permet le changement des critères d'évaluation par admission.

Données financières
  • Données financières peuvent être remplies manuellement ou intégrées avec un système financier.
  • Budget notionnel: les entités du Centre de responsabilité | A, du Centre de responsabilité | PG et du budget annuel affiche les données du budget du programme.
Points à considérer pour la conception
  • Les programmes doivent avoir une possibilité de financement pour accepter les projets.
  • Un programme enfant peut identifier le programme parent ultime comme un « programme réel » pour simplifier la déclaration de rapports.

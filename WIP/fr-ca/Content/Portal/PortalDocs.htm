Documents dans le portail

Dans le portail, les documents sont stockés dans des bibliothèques de documents SharePoint. Voir Intégration de documents du portail au GPSC pour un graphique de l'architecture.

Remarque:

  • Le contrôle de version est activé dans toutes les bibliothèques de documents. Télécharger une nouvelle copie d'un document ou modifier un document produira une nouvelle version.
  • Les bibliothèques de documents sont protégées par les permissions de site de SharePoint. En règle générale, les utilisateurs auront le même accès à une bibliothèque de documents qu'à la zone. Si un utilisateur ayant un accès en lecture a été invité à un client, il aura un accès en lecture à toutes les bibliothèques de documents de ce client ou à tous les projets du client. Si un utilisateur n'a pas accès à un client, il n'aura pas accès aux bibliothèques de documents du client ou aux projets du client.
  • Les documents peut être téléchargés à l'aide de l'interface Web de SharePoint ou en glissant-déplaçant les fichiers dans la fenêtre du navigateur pendant la visite de documents partagés.
  • Les bibliothèques de documents prennent en charge une structure de dossiers.
  • Les documents qui sont supprimés dans une bibliothèque de documents vont dans la corbeille de SharePoint. Les utilisateurs du portail peuvent explorer et récupérer les documents dans la corbeille en cas de suppression accidentelle.
  • SharePoint prend en charge l'intégration à Microsoft Office Online Server (OOS). Si SharePoint est relié à un serveur OOS, les utilisateurs pourront modifier directement les documents Office  (par exemple, Word, Excel) dans leur navigateur Web. Il s'agit d'une fonction facultative. Si OOS n'est pas présent, les utilisateurs devront télécharger, modifier puis charger de nouveau les documents pour apporter des changements.
  • Le fait de modifier des documents au moyen d'OOS crée de nouvelles versions. Le fait de charger de nouveau un document créera aussi une nouvelle version, pourvu que le nom du fichier n'ait pas été changé.

  • Les  URL des diverses bibliothèques de documents sont configurées dans la configuration EGCS.Portal.

    • Consulter Configuration du portail de la Solution d'entreprise pour les subventions et les contributions (SESC). Veuillez noter que le fait de changer des URL dans la configuration ne touche que les nouveaux sites – les sites existants ne seront pas touchés. Cependant, le code de portail tentera d'utiliser les nouveaux noms d'URL pour trouver les bibliothèques de documents, ce qui entraînera des liens brisés pour les sites existants.

    • Les noms d'URL devraient être établis avant de créer une zone client, une zone de projet ou une zone personnelle dans le portail et ne pas être changés par la suite.

Bibliothèques de document

Les bibliothèques de documents sont créées d'avance par le service de création de sites de Windows – voir Service de Windows : création de site

Chaque bibliothèque de documents a un but précis :

  • Documents partagé : Documents arbitraires qui sont chargés par des utilisateurs du portail afin de les échanger avec l'agence de développement régional (ADR) ou avec d'autres utilisateurs du portail pour le même client ou projet. Documents Word, feuilles de calcul, photos ou tout autre type de document.

    Cette bibliothèque de documents est visible sur la page maître du portail. L'utilisateur a toujours un lien à suivre pour les documents partagés, qui est contextuel selon la zone actuel (client, projet ou personnelle).

    EGCS.Portal configuration setting - DocumentLibraryPortalDocument

    Default URL - https://<portal>/<site>/Document

  • Présentations : Formulaires de SmartGuide d'Alphinat et fichiers « joints » à ces présentations de formulaires. Un dossier est créé automatiquement dans cette bibliothèque pour chaque formulaire créé par les utilisateurs de portail. Le fichier XML du formulaire SmartGuide, et tous les fichiers joints de ce formulaire, sont stockés dans le dossier du formulaire.

    Une fois qu'un formulaire a été présenté, les permissions de sécurité dans le dossier du formulaire sont changées pour que le contenu soit en lecture seule pour tous les utilisateurs. Cela empêche tout autre changement aux pièces jointes d'un formulaire ou d'un fichier.

    La page maître du portail n'a aucun lien vers cette bibliothèque de documents. La bibliothèque est visible au moment de modifier des formulaires SmartGuide et d'utiliser la fonction de pièce jointe de fichiers. La bibliothèque de documents n'est pas cachée; il n'y a tout simplement aucun lien pratique qui permette d'y accéder. Si un utilisateur connaît l'URL de la bibliothèque de documents, il peut choisir d'accéder a cette bibliothèque.

    EGCS.Portal configuration setting - DocumentLibraryPortalSubmission

    Default URL - https://<portal>/<site>/SubmissionSoumission

  • Registres : Documents sommaires, lettres ou autres registres formels pouvant être affichés dans le portail. Ces documents viennent de l'ADR et sont copiés dans le portail. Cette bibliothèque de documents en est lecture seule pour tous les utilisateurs du portail.

    La page maître du portail n'a aucun lien vers cette bibliothèque de documents. Diverses parties Web affichent des liens vers des documents sommaires et d'autres registres, au besoin. La bibliothèque de documents n'est pas cachée; il n'y a tout simplement aucun lien pratique qui permette d'y accéder. Si un utilisateur connaît l'URL de la bibliothèque de documents, il peut choisir d'accéder a cette bibliothèque.

    EGCS.Portal configuration setting - DocumentLibraryRdaRecord

    Default URL: https://<portal>/<site>/Record

Synchronisation des documents dans le GPSC

Le service de gestion des documents de Windows (voir Windows service - document management) recherche périodiquement le portail et l GPSC pour y trouver des documents nouveaux ou mis à jour. Ces documents sont copiés du portail a u GPSC, ou du GPSC au portail, au besoin.

Au moment d'établir une interface avec le GPSC, le portail communique toujours et uniquement avec le service Web d'ADR (voir Service Web des ADR ). Il utilise les méthodes du service DocumentManager.svc pour copier des documents de façon continue. Le service Web d'ADR est ensuite responsable de copier les documents dans le GPSC.

Le service Web DocumentManager.svc communique avec le GPSC au moyen d'un ensemble de méthodes d'interface de protocole d'application (API) publique au moment de synchroniser des documents.

Les présentations de portail sont prises en charge par le service Web FormManager.svc, qui utilise un ensemble distinct de méthodes d'API publique pour présenter les données et les pièces jointes dans le GPSC.

Voir Composantes de portail du GPSC pour des renseignements sur les méthodes d'API publique.

Documents dans le GPSC

Dans le GPSC, les documents sont stockés dans un dépôt interne de documents, qui est probablement GCDocs ou  SharePoint.

  • Les documents qui se trouvent dans  GCDocs peuvent être déplacés librement et peuvent ne pas se trouver dans l'emplacement d'origine où ils ont été chargés. SharePoint n'offre pas cette capacité sans briser le lien avec le document.
  • Le contrôle de version est pris en charge par le dépôt interne de documents.

  • Un document qui se trouve dans le portail a deux copies :

    1. La copie officielle stockée dans le dépôt interne de documents du GPSC et
    2. La copie de commodité stockée dans SharePoint, dans le portail.

  • Les documents qui se trouve dans le GPSC et qui viennent du portail ne peuvent être modifiés que dans le portail, et non dans le GPSC. Toute modification apportée directement aux documents du portail dans le GPSC ne sera pas prise en charge dans le portail, c'est-à-dire les copies du GPSC sont en lecture seule.

  • Le GPSC utilise le plugiciel de gestionnaire de documents développé sur mesure pour toutes les interactions entre le GPSC et le dépôt interne de documents.

  • Les configurations suivantes des emplacements de documents doivent être configurées pour les entités  : compte, utilisateur du portail et projet. Le nom du dossier n'est pas important.

    • Documents du portail (Portal Documents) : Ce dossier stocke les documents que le client charge dans le portail.

    • Présentations du portail (Portal Submissions) : Ce dossier stocke les documents liés à une présentation de formulaire dans le portail, par exemple, des données de présentation, le document sommaire d'une présentation, les pièces jointes d'une présentation.

  • Les documents des registres des ADR sont synchronisés avec le portail chaque fois qu'un document qui se trouve dans le GPSC est signalé comme un document de portail  ET est associé à un compte, un utilisateur de portail ou un projet.
  • Avant de créer un site de portail pour un client ou un projet, les agents du GPSC peuvent signaler d'avance que des documents sont des documents de portail, ce qui les synchronisera avec le portail lorsque le site de portail est créé.
  • Lorsqu'un document du portail est ajouté pendant la synchronisation, le GPSC reproduira la structure de dossier du document. Cela empêche que des documents portant le même nom s'écrasent.

About the transition framework

The transition framework manages the life cycle of business records (e.g. project life cycle, disbursement review, etc.) and automates some aspects of business process flows as records change status. Transitions identify pathways between steps in a business process flow.Your organization can configure unique processes for programs, amendments, claims, and many others. Transition templates can be reused across multiple programs if a generic process is required.

Transition Framework functionality highlights:

  • Transitions are linked to a BPF and bundled in a template that can be applied to entities through the program profile.
  • The transition framework can be customized to your organization's unique business process.
  • The transition framework works with:
    • The Action framework to automate actions in each transition.
    • The Authority framework to enforce an approval chain of one or multiple teams before allowing a transition to take place. If a transition requires a decision from a delegated authority, the approval pathway can be configured in the transition (e.g. manager / director / director-general / Assistant Deputy Minister / Deputy Minister) before sending to the Minister for approval, to complete a transition).
    • The Decision framework to record decisions and generate decision-related documents.

Use transitions to configure what happens when a users clicks Make Transition, for example: record decisions, capture snapshots of data, lock records, change the status of a project. Each transition captures the date and time of the transition, as well as other factors that have been configured.

A single set of transitions can be used across multiple programs, or each program can have a tailored set of transitions.

Transition framework components include:

  • Transition templates are the principal configuration entity for the framework. One or more transition paths are related to a transition template.
  • Transition paths contain the logic that defines each pathways in a business process flow. Transition paths can be configured to include validations, decisions, actions and approvals.

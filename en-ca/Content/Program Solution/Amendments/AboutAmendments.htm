About amendments

Amendment feature - in brief

Amendments is a GCPM tool to safeguard the integrity of contribution agreements (CAs) over time. The amendment feature locks / unlocks GCPM fields that appear in a CA after an assessment has been approved. Amendments can be made during the CA negotiation stage, as well as the project administration stage. Amendments are used for formal amendments as well as administrative / internal changes related to the project that require modifying the fields displayed in the CA.

GCPM includes a number of pre-configured amendment types (e.g. administrative amendment, substantive 2-way amendment, etc. as well as default). For each type of amendment, a specific set of transitions, validations and approval path has been designed. Data integrity of the project is preserved: an amendment does not overwrite data until approval has been granted for the changes.

Amendments are captured with a snapshot; these are available in the amendment grid.

Automation
  • Data is automatically copies to the project when an amendment is finalized. No additional steps required by users.
  • Hide \ show of amendable items tabs will ensure users amend all pertinent items related to a type of amendment.
Design considerations:
  • Amendments are managed in a separate entity, and data is copied back and forth between amendment and project entities.
  • There are no saved copies of amendments. Once an amendment is finalized, a snapshot is created. A user cannot see previous amendments in the project profile.

Associated tools
Transitions - in brief

Transitions is a configuration tool, used in conjunction with business process flows and actions, to manage the life cycle of business records (e.g. project life cycle, disbursement review, etc). Your organization can use transitions to configure when to: record decisions, capture snapshots of data (at a moment in time) , lock records, and many other automated actions.

Risk assessments can be generated as part of the project life cycle, (e.g. integrated into the assessment process) without requiring officers to navigate and undertake risk as a separate activity.

The GCPM XX (name) includes transitions related the negotiation and execution of a CA.

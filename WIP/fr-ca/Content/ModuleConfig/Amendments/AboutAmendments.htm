À propos des modifications

Les modifications protègent l'intégrité des ententes de contribution (EC) au fil du temps. La fonctionnalité de modification comprend le verrouillage et le déverrouillage des champs du GSPC qui s'afficher dans une EC. Les modifications officielles et administratives peuvent avoir lieu pendant les négociations de l'EC et les étapes d'administrations du projet.

Le GSPC comprend un nombre de types de modification pré-configuré (par exemple, modification administrative, amendement de fond bidirectionnel, par défaut, etc.). Chaque type de modification a un ensemble de voies de transition, de validation et d'approbations liées. Lorsqu'un utilisateur choisi un type de modification, le GSPC affiche tous les champs modifiables nécessaires sur le formulaire de modification.

Les catégories de modifications regroupent les éléments de modification appropriés ensemble. Par exemple, une modification du coût comprend des onglets avec des éléments modifiable liés aux coûts du projet, aux flux de trésorerie, à l'horaire de remboursement et au financement. La liste des catégories de modification peut être facilement modifiée par votre organisation.

Une modification ne remplace pas les données du projet avant l'approbation des changements, ce qui assure l'intégrité des données. Tous les éléments modifiables sont affichés selon l'évaluation du projet et affiche une version verrouillé « actuelle » des données, sans prendre en compte la version modifiable. À l'approbation, les données sont automatiquement copiées au projet. Aucune étapes additionnelle n'est requise par les utilisateurs.

Les modifications sont saisies avec une capture d'écran et affiché sur la grille de modification du projet.

Votre organisation peut:

  • Choisir quand les modifications se produisent, selon le cycle de vie du projet pour tout programme.
  • Choisir qui amorce les modifications.
  • Modifier les pouvoirs d'approbation, les validations, la génération de documents, etc. associés aux modifications afin de s'adapter à vos exigences.

Catégorie Détails

Données de projet clés
  • Les données particulière à un projet peuvent être modifiées (par exemple, coûts, flux de trésorerie, dates, etc.).
  • Le GSPC offre une liste de champs modifiables communs.
  • La configuration de champs modifiables additionnels requiert l'utilisation d'un nombre d'outils de configuration (clone, grille, validations, etc.).

Objectif de la modification
  • Une grille configurable est offerte pour les ADR afin de saisir ;es objectifs de modification pour faciliter la déclaration de rapports.
  • Aucun outil de déclaration de rapports n'a été configuré.

Décision de modifications
  • Les décisions liées aux transitions de modification configurées sont faciles d'accès.
  • Configurer les décisions pour correspondre aux processus de votre organisation en utilisant l'outil de transition.
Points à considérer pour la conception:
  • Les modifications sont gérées dans une entité séparée et les données sont copiées entre les entités de modification et de projet.
  • II n'y a aucune copies enregistrées des modifications. Au moment qu'une modification est terminée, une capture d'écran est générée.

  • Le plugiciel clone copie les données modifiées dans les champs du projet appropriés. Chaque champ modifiable doit être configuré avec le plugiciel clone.

Arriéré et conception future

  • Actuellement, il n'y a aucun mécanisme automatisé qui empêche les utilisateurs de modifier les données dans un onglet, puis à désélectionner l'onglet pour que cela ne s'affiche pas sur le formulaire.

  • Lorsqu'une modification est terminée, un nouvel EC est généré manuellement. Il n'est pas généré automatiquement.

  • Les modifications peuvent seulement modifiées les renseignements dans un projet associé et non dans un enregistrement du client.

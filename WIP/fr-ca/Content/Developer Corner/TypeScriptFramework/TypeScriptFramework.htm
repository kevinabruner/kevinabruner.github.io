Cadre (pour TypeScript)

main.js

Analogue la méthode « principale statique » dans d'autres langues, ce fichier est le principal point d'entrée pour tous le sscripts JS.

Il y a trois endroits dans CRM où JavaScript peut être utilisé.

  • Formulaire
  • Barre du ruban
  • Formulaire,  Grille, Sous-grille, Affichage associé,  Tableau de bord, etc.
  • Ressource Web HTML

Par conséquent, il y a trois « principaux » points d'entrée, énuméré ci-dessous. Ces points d'entrée peuvent être considéré comme la classe de « base » qui établit tout pour le script.

Principal point d'entrée pour les Scripts de formulaireEGCS.Main.Form.load()

Configuration :

  • Script: egcs_/shared/js/main.js
  • Form Event: On Load
  • Form Function: EGCS.CRM.Form.load
  • Paramètres
  • Chemin vers le fichier de script
  • Autres paramètres requis

Un module de formulaire devrait seulement exporter la fonction par défaut, laquelle recevra les valeurs d'entrée suivantes.

  • FormContext objet
  • Objet des événements
  • Toutes les valeurs précisées dans l'éditeur du formulaire comme paramètres d'entrée séparés.
Principal point d'entrée pour les commandes de la barre du ruban –EGCS.Main.Command.load()

Configuration :

  • Créez un DisplayRule or EnableRule pour télécharger le script de commande.
  • Ressource Web : egcs_/shared/js/main.js
  • Fonction: EGCS.Main.Command.load
  • Premier paramètre d'entrée : « chemin au script de commande »
  • Inclure facultativement la fonction d'appeler comme fil de requête
  • Passez toutes les autres valeurs de paramètre comme désiré.
  • Cela téléchargera toutes les exportations dans l'espace de nommage EGCS.Main.Command
  • Faites attention aux doublons
  • Créez une Action de ruban afin de faire appel à la fonction correspondante dans le script de commande.
  • Ressource Web : egcs_/shared/js/main.js
  • Fonction: EGCS.Main.Command.<function_name>
  • Doit passer PrimaryControl comme le premier paramètre d'entrée.
  • Passez toutes les autres valeurs de paramètre comme désiré.
Principal point d'entrée pour les ressources Web  HTML –EGCS.Main.Page.load()

Dans le HTML, télécharger, en ordre :

  • <script src=“ClientGlobalContext.js.aspx” />
  • <script src=“egcs_/shared/js/main.js” onload=“EGCS.Main.Page.load(‘path to module’)” />

Un module de page devrait seulement exporter la fonction par défaut, laquelle recevra les valeurs d'entrée suivantes.

  • Objet PageContext

Toutes les dépendances du script devraient être téléchargées à l'intérieur du fichier du script en utilisant l'importation et l'exportation.

context.js

C'est l'objet de contexte qui se fait passer aux gestionnaires d'événement, comme onload, onsave, onchange, etc.

Tout comme le cadre du côté serveur, il y différents types d'objets de contexte :

  • Context
  • C'est la classe du contexte de base, laquelle est une classe abstraite.
  • FormContext
  • C'est l'objet de contexte pour les scripts de formulaire.
  • Contient des objets wrapper pour certains objets XRM.
  • CommandContext
  • C'est l'objet de contexte pour les commandes de la barre du ruban.
  • Il contient aussi le contexte du formulaire si c'est le formulaire de la barre du ruban.
  • PageContext
  • C'est l'objet de contexte pour les ressources Web HTML.
  • Il contient aussi le contexte du formulaire si c'est téléchargé à partir d'un formulaire.

events.js

L'objet des événements est seulement utilisé dans les scripts de formulaire d'entité.

Similaires à la classe PluginExecution dans le cadre du côté serveur.

Il contient tous les événements de formulaire possible auxquels vous pouvez joindre un gestionnaire d'événement.

Il contient des événements personnalisés qui n'existe pas dans CRM, p.ex. :oncreate, onupdate, etc.

core.js

Un ensemble de classes de base utilisé par le cadre du coté client.

Le cadre ne fonctionnera pas sans ces classes.

common.js

Il comprend tous les codes communs et partagés. Il est utilisé par la plupart des ressources Web  JavaScript.

Éventuellement nous les séparerons en modules appropriés. Par contre, les modules seront encore rassemblé en une seul fichier .js avec le nom « common.js » ou « shared.js » ou un nom similaire.

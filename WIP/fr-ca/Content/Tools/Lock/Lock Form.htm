Configurer un formulaire pour afficher les champs verrouillés

Afin d'améliorer l'expérience de l'utilisateur, les formulaire peuvent afficher un icône de verrou sur les champs qui sont verrouillés. Les champs verrouillés ne peuvent pas être modifiés jusqu'à ce que le verrou soit retiré.

Au moment de la publication de GPSC, les entités suivantes ont été configurée afin d'afficher des icônes de verrou lorsque l'enregistrement est verrouillé:

Pour configurer un formulaire afin qu'il affiche une icône de verrou:

  1. Aller au formulaire sur lequel vous voulez afficher l'icône de verrou et cliquez sur Propriétés du formulaire:

    • S'il y a une référence de la bibliothèque du formulaire à egcs_/shared/js/main.js, le formulaire est déjà configuré.

    • S'il n'y aucune référence, ce formulaire n'est pas configuré (d'autres formulaires pour cette entités pourraient l'être) ou cette entité n'est pas encore configurée.

  2. Pour déterminer si cette entité a été configurée pour utiliser le cadre JavaScript du  GPSC, allez à la solution GCPM_JS solution and look for a web resource with the name egcs_/entities/<entity name>/forms/default.js.

  3. Si nul n'existe, créez une ressource Web dans un format TypeScript. Il fera une référence au import { LockField } from egcs_/shared/js/lock";".
  4. Consulter les fichiers pour toutes les entités mentionnée ci-dessus à titre d'exemple.

Clause de non-responsabilité: La création d'un nouveau fichier javascript ou de tout nouveau code, côté serveur ou autre, doit être discutée avec le cluster GPSC, dans la mesure où cela pourrait devenir une fonctionnalité partagée intégrée à une future version de GPSC. Ceci sera mis à jour une fois que la méthode pour contacter le projet post-transition de l'équipe GPSC sera déterminée.

Grilles Crowe

Les grilles CROWE sur le formulaire à être verrouillé ont besoin d'une bibliothèque JavaScript pour définir les colonnes de champs verrouillés en lecture seule pour les enregistrements verrouillés, afin de s'assurer que l'utilisateur puisse seulement modifier les champs qui sont déverrouillés dans la grille. La colonne du champ de verrouillage de l'entité doit être inclus, même s'il n'est pas visible.

La fonction lockFieldOnGrid utilise un paramètre avec le nom de l'entité de la grille qui est passé en utilisant un argument spécialisé.

Remarque: ces configurations sont seulement pour une amélioration de l'IU. Si ce n'est pas mise en œuvre et qu'un utilisateur tente de modifier un enregistrement verrouillé, le système empêchera la tentative d'enregistrement par un message d'erreur à l'utilisateur.

Pour s'assurer que les utilisateurs peuvent seulement modifier les champs qui sont déverrouillé (une fois affiché dans une grille):

  1. Ouvrir le formulaire qui affiche les valeurs verrouillées dans une grille CROWE.
  2. Verrouiller les grilles CROWE sur le formulaire. Cela requiert une bibliothèque JavaScript pour fixer les colonnes des champs verrouillés en lecture seule pour les enregistrements verrouillée (voir le tableau ci-dessus).
  3. Assurez-vous d'inclure la colonne de champ verrouillé de l'entité, même si elle n'est pas visible.
  4. Enregistrez et fermez le formulaire.

Paramètres de la grille sur le formulaire:

  • Sélectionnez le code de type d'objet d'enregistrement d'option de grille et l'identifiant unique en tant que paramètres.
  • Argument personnalisé:config=Grid Name&entity=egcs_entityName

Prenez connaissance de toutes autres fonctions sur le formulaire qui pourraient avoir besoin d'être modifié une fois que le plugiciel de verrouillage est mis en œuvre. Le script du egcs_LockMetadata peut être utile pour récupérer les renseignements du CFG Verrouillage et CFG Champ de verrouillage ainsi que pour évaluer si l'enregistrement est verrouillé.

Exemple :

  • Vérifiez les modifications au bouton de la barre d'outils séparée de la grille FC Disbursement Review FC Claim Item pour empêcher la séparation d'un Examen de réclamation d'un examen de débours verrouillé.

Application Web des extensions des services ADFS (Active Directory Federation Services)

Les extensions ADFS sont une application Web qui élargissent les ADFS de manière à répondre aux exigences de la fédération des justificatifs du gouvernement du Canada (FJGC), selon ce qui est documenté dans la spécification des Solutions technologiques d'authentification électronique (STAE)*. Les extensions ADFS offrent la fonctionnalité suivante :

  • Un point de terminaison pour recevoir les notifications de déconnexion de canal de la FJGC au moyen de messages SAML SOAP.
  • Une liaison faisant en sorte que la déconnexion à partir du portail puisse être signalée à un fournisseur de la FJGC à l'aide du protocole SAML SOAP, au lieu de l'ancien protocole HTTP‑Redirect.
  • Transposer l'information sur les sessions ADFS dans le portail au moment de l'ouverture de session.
  • Supprimer l'information sur la session ADFS au moment de la fermeture de session, lorsque la fermeture de session n'est pas traitée au moyen des ADFS.

Remarque : Les extensions ADFS ne sont pas prises en charge par Microsoft.

Conditions préalables

  • IIS doit être installé sur le serveur.
    Add-WindowsFeature Web-Server
    Add-WindowsFeature Web-WebServer -IncludeAllSubFeature
    Add-WindowsFeature Web-Mgmt-Tools –IncludeAllSubFeature
  • Une version du projet GC.GCCF, qui, suppose‑t‑on, se trouve dans un répertoire « _Build »

Configuration – Installateur

Cette configuration définit comment et où l'application Web des extensions ADFS est installée. Le fichier de configuration se trouve dans le projet « Configuration » GC.GCCF.

GC.GCCF\<Configuration project>\Config\<portal name>.config.xml

Exemple de configuration (serveur d'intégration)  :

GC.GCCF\Configuration EGCS-SESC\Config\integration.config.xml

  • CertificateSubjectSsl: Indiquer le sujet de certificat du certificat SSL pour les ADFS. Le code d'installateur cherchera les certificats dans le magasin personnel de l'ordinateur local pour y trouver un certificat avec un sujet correspondant.
  • WebAppIpAddress: Indiquer l'adresse IP à laquelle l'application  Web devrait être liée dans IIS.

  • AdfsServiceAccount: Indiquer le nom du compte de service pour ADFS L'application Web des extensions ADFS sera lancée sous ce compte.

Configuration – FJGC

Cette configuration définit comment les extensions ADFS sont reliées à l'infrastructure de la FJGC. Cette information est en grande partie semblable à l'information configurée dans les ADFS, puisque les extensions ADFS doivent se connecter aux serveurs de la FJGC comme s'il s'agissait Ces fichiers de configuration se trouvent dans le projet « Configuration » de la FJGC.

GC.GCCF\<Configuration project>\Config\GccfAdfsExtension\<portal name>\

Exemple de configuration (serveur d'intégration)  :

GC.GCCF\Configuration EGCS-SESC\Config\GccfAdfsExtensions\integration

Fichiers

  • fedlet.cot – sun-fm-trusted-providers should contain the entityID of the portal, GCKey, and SecureKey. The entityID is found in the metadata file for each of those.
  • idp1-extended.xml – entityID doit correspondre aux métadonnées de la SecureKey
  • idp1.xml – Les métadonnées actuelles de la  SecureKey telles qu'elles sont reçues de Services partagés Canada (SPC)
  • idp2-extended.xml – entityID doit correspondre aux métadonnées de la CléGC
  • idp2.xml – Les métadonnées actuelles de la CléGC telles qu'elles sont reçues de SPC
  • sp-extended.xml – entityID must match the portal metadata.
    signingCertThumbprint doit contenir l'empreinte du certificat de signature du portail.
  • sp.xml – entityID doit correspondre aux métadonnées du portail

Installation

Les étapes ci‑dessous partent du principe que l'application  Web de domaine commun de la FJGC est installée sur le serveur des ADFS.

  1. Ajouter une fonction de Windows Identity Foundation :
    Add-WindowsFeature Windows-Identity-Foundation
  2. Create a new web application for the ADFS Extensions web application

    .\_Build\Scripts\Create-GccfAdfsExtensions.ps1

    By default, this will create:

    • Web application “GCCF ADFS Extensions”
    • Application pool “GccfAdfsExtensions” using the ADFS service account
    • Path C:\inetpub\wwwroot_gccf_adfs
  3. Install the ADFS Extensions web application

    .\_Build\Scripts\Install-GccfAdfsExtensions.ps1

    This will install the web application to:
    C:\inetpub\wwwroot_gccf_adfs\AdfsExtensions

  4. Install configuration files for the ADFS extensions web application

    .\_Build\Scripts\Install-GccfAdfsExtensions-Config.ps1

    This will install configuration files to:

    C:\inetpub\wwwroot_gccf_adfs\AdfsExtensions\App_Data

  5. Installer le gccf.config fichier de configuration

    .\_Build\Scripts\Install-GCCF_Config.ps

Remarques :

  • L'application Web des extensions des ADFS est développée à partir d'un cadre SAML à source ouverte, ForgeRock OpenAM Community Edition. De légères modifications et extensions ont été apportées au cadre pour le relier à la solution de connexion de la FJGC, ainsi que corriger quelques bogues ou lacunes des fonctions. Cependant, le cadre est responsable d'envoyer et de recevoir les messages SAML.
  • Le cadre OpenAM se trouve dans le projet « OpenAM Community Edition » dans la GC.GCCF solution.
  • Voir ces remarques sur github* pour plus de détails.

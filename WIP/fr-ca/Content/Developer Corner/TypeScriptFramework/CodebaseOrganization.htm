Organisation du code base

Structure

La structure du cadre requiert que tous les noms de ressource Web sont du chemin du egcs_ et comprennent l'extension.

Le dossier racine virtuel dans CRM estegcs_

egcs_/actions Contains early bound classes of CRM custom actions.

egcs_/entities

Contient des ressources Web particulières, p. ex. des scripts de formulaire, des scripts de ruban, des classes liées au préalable, etc.

La structure dans ce dossier est:

Entité de ressource Web

Commentaires
egcs_/entities/<entity_name>/forms

Un script par formulaire

Puisque la plupart des entités ont seulement un formulaire, la convention d'affectation des noms estdefault.ts

egcs_/entities/<entity_name>/pages

Entité des ressources Web HTML particulières

egcs_/entities/<entity_name>/ribbon

Puisque actuellement seulement un fichier est nécessaire pour l'entité des commandes de ruban, la convention d'affectation de noms estcommands.ts

egcs_/library

Contient des bibliothèques de tiers partis, p. ex. bootstrap, jQuery, etc. La structure se réfère à la structure de la bibliothèque de tiers parti, s'il y en a une. Si c'est un type de fichier unique, la dernière version du dossier n'est pas nécessaire, simplement ajouter le numéro de la version au nom du fichier et laisser le numéro de la version pour la dernière version.

La structure dans ce dossier est:

  • egcs_/library/<name>
  • egcs_/library/<name>/latest
  • egcs_/library/<name>/latest/<structure>
  • egcs_/library/<name>/<version>
  • egcs_/library/<name>/<version>/<structure>

egcs_/shared

Contient des ressources Web générales et réutilisables, p. ex. CSS, JS, HTML, polices, images, données (XML et  JSON), ressources (resx).

La structure dans ce dossier est:

  • egcs_/shared/css
  • egcs_/shared/data
  • egcs_/shared/data/xml
  • egcs_/shared/data/json
  • egcs_/shared/fonts
  • egcs_/shared/html
  • egcs_/shared/images
  • egcs_/shared/js
  • egcs_/shared/resources
  • Ce sont des fichiers .resx (lesquels sont convertis en JSON pour la version CRM <9)

egcs_/html

Contient des ressources Web  HTML particulières et des ressources liées au Web, p. ex. CSS, JS, HTML.

La structure dans ce dossier est:

  • egcs_/html/<name>/index.html (s'il n'y en a seulement une)
  • egcs_/html/<name>/css
  • egcs_/html/<name>/css/styles.css (s'il n'y en a seulement une)
  • egcs_/html/<name>/data
  • egcs_/html/<name>/fonts
  • egcs_/html/<name>/images
  • egcs_/html/<name>/js
  • egcs_/html/<name>/js/script.js (s'il n'y en a seulement une)
  • egcs_/html/<name>/resources

Cette structure s'applique aussi a :egcs_/shared/html and egcs_/entities/<entity>/pages

Généralement, les ressources Web HTML sont autonomes avec leurs propres fichiers et suivent la structure egcs_/shared.

Contrôle des versions

Le besoin de contrôler les versions se produit rarement, à l’exception des bibliothèques de tiers partis.

Pour des ressources Web personnalisées :

  • La dernière version qui est mise à jour ne devrait pas contenir un numéro de version dans le nom du fichier.
  • S'il y a un grand changement et que les deux versions ont besoin d'être soutenues, créez une copie de la ressource Web et ajoutez un numéro de version au nom du fichier, puis mettez à jour la dernière version.
  • Si c'est pour le contrôle des versions d'une ressources Web HTML et ses fichiers liés, créez un nouveau dossier pour le version mise à jour et un dossier des versions pour les versions précédentes.

Pour les bibliothèques de tiers partis, nous utilisons :

  • Si elle contient un seul type de fichier, p. ex. seulement js ou CSS, suivez la règle ci-dessus.
  • Dans le cas contraire, suivez la règle ci-dessus pour le contrôle des version des ressources Web HTML.

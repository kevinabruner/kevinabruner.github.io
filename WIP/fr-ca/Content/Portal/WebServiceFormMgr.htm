Service Web FormManager

La classe de service Web FormManager comporte de nombreux services Web qui communiquent avec le portail ou avec le CRM au moyen des appels de service Web de l'ADR des formulaires SmartGuide.

Les directives sur l'ajout de références de services Web à SmartGuide sont accessibles au Conception de formulaires pour le portail avec SmartGuide.

Services Web utilisés dans les formulaires SmartGuide

Services Web utilisés dans les formulaires SmartGuide :

Retours

LoadDraft          

chaîne ReferenceNumber,

chaîne xml

Au lancement du Smartlet, interrogez l'ensemble de documents SubmissionSoumission de l'instance de formulaire actuelle relativement au XML pour la désérialisation et le remplissage des champs Smartlet.

SaveDraft          

chaîne ReferenceNumber,

chaîne FormXml,

chaîne FormPdfBase64,

bool IsSubmit

résultat bool.

On Save button click or Master Page language toggle sends the serialized FormXml and referenceNumber to the current form instance’s SubmissionSoumission document set. FormPdfBase64 is blank and IsSubmit is set to false. Returned results triggers the SaveResult field to inform the user of success or failure.

On Submit button click the generated FormPdfBase64 encoded string is included and IsSubmit is toggled from false to true. If returned result is true advances the user to Successful Submission page that displays their ReferenceNumber

UserCanEditForm         

 

résultat bool.

Au lancement du Smartlet, indiquez si l'utilisateur actuel a, ou non, la permission appropriée pour sauvegarder et soumettre le formulaire.

GetProjectOutcomes          

 string FC_Profile_ID

Data.FCProfile.egcs_fc_result[]

Au lancement de Smartlet de la déclaration de la mesure du rendement, le Smartlet appelle le service Web GetProjectOutcomes web service which queries the RDA web service GetProjectOutcomes du portail qui interroge le service Web de l'ADR et retourne un tableau d'objets de données qui tiennent compte des résultats de projet qui sont accessibles au profil d'analyse du financement dont l'identifiant a été fourni.

 

GetProgramsList          

 

Data.Program.egcs_TP_Profile[]

N'est actuellement pas utilisé dans les formulaires, mais fournit une liste de programmes actuellement accessibles au cas où l'on souhaiterait que le Smartlet retire les données du CRM au lieu de les coder en dur dans un menu déroulant Smartlet.

GetFCProfileID          

 

FCProfileID

Sur les formulaires au niveau du projet (réclamations, avances et formulaires de déclaration de la mesure du rendement), le lancement de Smartlet récupère l'identifiant de profil de l'analyse de financement.
Remarque : Retourné comme un FCProfileID and not a string as SmartGuide has its SharePointConnections configuré en tant que valeur de retour JSON.

GetBudgetCategories           

string FC_Profile_ID

Data.Claim.egcs_FC_BudgetItem[]

Au lancement de Smartlet sur les réclamations et les formulaires d'avance, interrogez le service Web de l'ADR GetBudgetCategories , qui retourne un tableau d'objets de données utilisées pour remplir les options de menu déroulant de la catégorie du budget disponible, respectivement, dans les tableaux des items de réclamation et des items d'avance des formulaires de réclamation et d'avance.

GetProgramQuestionsFromCRM

Guid applicationFormIdGUID

Data.FormManager.egcs_ApplicationForm

Dans le formulaire d'application dynamique, au moment de basculer de la page 1 a la page 2, l'appel de formulaire GetProgramQuestionsFromCRM which passes the supplied GUID applicationFormIdGUID (which is set by the Program drop down on Page 1) to the RDA web service GetProgramSpecificQuestions si l'actuel programme de formulaire a fait l'objet de modifications, ce qui exige la mise à jour des questions dynamiques du CRM qui sont affichées à la page 2.

L'objet retourné comporte un tableau de questions et les métadonnées appropriées pour construire et modifier les SmartletFields, de même que le GUID représentant le modèle retourné au formulaire.

CreateDraft

int FormDefinitionId

résultat de la chaîne

Called from the Master Page Create dropdown, creates an instance of a form whose type is determined by the provided FormDefinitionId in the current portal scope. The returned result is the URL that the page is redirected to which renders the newly created draft in SmartletRender.aspx

RemoveDraft           

chaîne ReferenceNumber,

SubmissionRemoveResult

Appelé de la partie Web des ébauches à partir de la page d'activités récentes, élimine l'ébauche correspondant au referencenumber ayant été fourni.

ListDraft
           

 

SubmissionListDraftResult

Appelé de la partie Web des ébauches à partir de la page d'activités récentes, énumère tous les formulaires d'ébauche accessibles au Web actuel. Les renseignements fournis comprennent les suivants : numéro de référence, type, données crées, création par, données modifiées, modification par.  En plus des liens pour modifier ou éliminer l'ébauche.

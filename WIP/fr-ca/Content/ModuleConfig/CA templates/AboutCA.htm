À propos des ententes de contribution

Lorsque les clauses et les ententes de contribution (EC) sont configurées pour la GPSC, les utilisateurs peuvent générer en un simple clic une EC formatée pour un projet approuvé. Les documents peuvent être stockés dans le référentiel de gestion des documents. Il pourrait prendre quelques moments à être généré si un grand nombre de champs sont affichés dans le document.

Une entente de contribution peut comprendre :

  • Des clauses normalisées, des conditions spéciales (modifiables par un agent) et des clauses restrictives.
    • Les clauses normalisées et restrictives sont générées dans un modèle de document (voir le Générateur de documents, ci-dessous).
    • Des conditions spéciales sont gérées avec la fonction des clauses.
  • Toutes données stockées sur le projet et les entités liées peuvent être inclus dans l'entente de contribution. Cela comprend :
    • Les indicateurs de rendement et des objectifs
    • Les coûts du projet et les flux de trésorerie
      Pour exemple, une question oui ou non à propos des paiements anticipés peut insérer la clause pertinentes dans l'entente de contribution.
    • Le financement
    • Les questions d'évaluation, et les résultats
    • Les renseignements sur le compte (p. ex., nom légal, nom commercial). Les contacts affiliés au compte peuvent être identifiés dans l'EC au besoin.
    • Dates-clés.

Une fois qu'un projet est approuvé, les champs qui sont copiées dans l'entente sont verrouillés. Il peuvent seulement être modifiés (déverrouillés) en apportant une modification, afin de maintenir l'intégrité des données.

Les modèles de clause et les modèles de documents pour les ententes de contribution sont générés par un administrateur de GPSC.

Divulgation proactive : Une fois qu'une entente de contribution est exécuté, la GPSC peut générer un fichier de données du gouvernement ouvert, avec toutes les données nécessaires en vertu des règles actuelles de divulgation proactive (mars 2019). Des modifications peuvent être apportées pour des changements importants qui pourraient déclencher une nouvelle divulgation de données modifiées.

La GPSC peut générer des documents automatiquement pendant les transition, et les stocker automatiquement dans le référentiel de gestion de documents. Cela nécessite ce qui suit :

  • Les documents sont générés automatiquement basé sur les modèles XML. Consulter À propos du générateur de documents.
  • Votre organisation a intégré la GPSC dans son référentiel de documents. Voir À propos de la solution de gestion de document
  • Les ensembles de transitions, de décisions et de mesures sont configurés afin de générer et de stocker des documents dans le référentiel. Consulter À propos du cadre de transition.
Décisions de conception
Fonction de clause
  • La fonction de clause est conçue pour gérer les clauses qui nécessitent une formulation propre à un projet. La plupart des clauses sont stockées dans un modèle de documents.
  • Votre organisation peut mettre à profit les catégories de clauses afin d'accroître le nombre de clauses gérées avec la fonction de clause au lieu de le faire dans un modèle de document.
  • Votre organisation doit développer et maintenir au moins deux modèles de documents d'EC : un pour chaque langue officielle.
Générateur de documents
  • Les clauses appropriées sont affichées automatiquement dans la bonne section de l'EC, avec la numérotation appropriée.
  • Microsoft Word (version 2016 ou plus) supporte le XML. Aucun plugiciel supplémentaire n'est requis. Nécessite une connaissance de la programmation XML afin de créer des documents de sortie.
  • La GPSC génère des ententes de contribution dans Microsoft Word. Votre organisation peut mettre à profit des outils afin de générer des copies PDF, au besoin.
  • La GPSC n'inclus pas de mesures afin de créer automatiquement un PDF non modifiable. Votre organisation peut choisir de configurer cela différemment.
  • L'exemple d'entente de contribution de la GPSC affiche le budget du projet et les flux de trésorerie dans une grille dans l'annexe de l'EC. Il est à noter que les tableaux dans Word ne calculent pas automatiquement le sous-total des montants. Vous devez rafraîchir manuellement le document pour que les sous-totaux s'affichent.

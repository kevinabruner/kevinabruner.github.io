Financial integration Rest API

Financial System Rest API

The financial system REST API provides every endpoint that communicates to and from GCPM workflow steps. The API currently writes to an in-memory database and clients of the API are required to implement each endpoint to communicate with their financial system. It is important to note that when implementing each end point, the payload (or the model) should be the same as whatâ€™s implemented in the API. The API ships with a Swagger endpoint which illustrates each endpoint and what payload it requires. For example, the GCPM financial system REST API used for development and testing can be found here: https://gcpm-finapi-app.egcs-sesc.ca/index.html

Models

The following models are created in .NET Core using the entity framework. The entity framework can connect to the majority of major SQL database providers such as MSSQL, Oracle, PostgresSQL, MySQL or an in memory data store such as the implementation used during GCPM development and testing. Implementations must adhere to this model to ensure that the communication with CRM GCPM entities and workflows work out of the box.

Endpoints

The endpoints provided by the API will work out of the box provided that the implementation uses a database that adheres to the model. The integration with the financial system and therefore happen at a separate layer (e.g. leverage an ETL to move data to and from the financial system to the GCPM Financial System API database).
